1. Getting Started

System Requirements

Browser Chrome 90+, Firefox 88+, Edge 90+, Brave 1.25+
Backend Python 3.9+, PyTorch 2.0+, CUDA 11.8 (optional, for GPU)
Memory 8 GB RAM minimum, 16 GB recommended
Storage 2 GB for models, additional space for patient data

Quick Start

# Clone the repository
git clone https://github.com/zray-medical/zray-platform.git
cd zray-platform

# Install dependencies
pip install -r requirements.txt

# Download pre-trained models
python scripts/download_models.py

# Start the backend server
python manage.py runserver

# Open the web UI
open http://localhost:8080

Default Credentials

Username admin
Password admin

⚠️ Change default credentials immediately after first login in a production environment.

2. User Guide

Login & Authentication

Navigate to the login page and enter your credentials. Upon successful authentication, you'll be redirected to the Analysis Selector page where you can choose between X-Ray or ECG analysis. Sessions are stored in the browser and expire when you sign out.

Navigating the Platform

Page Purpose
Selector Choose analysis type from card-based overview
Dashboard View aggregate stats: total analyses, weekly count, avg confidence
X-Ray / ECG Upload medical images/data and run AI analysis
History Browse all past analysis records with filters
About Platform info, version, and contact details

Uploading Files

  1. Navigate to the desired analysis page (X-Ray or ECG).
  2. Click the upload zone or drag-and-drop your file.
  3. Wait for the upload and pre-processing to complete.
  4. The AI model will automatically run and display results.
  5. Review the diagnosis, confidence score, and annotated output.
  6. Results are saved to History for future reference.

3. Analysis Modules

X-Ray Analysis

Model ResNet-50 + Feature Pyramid Network
Training Data 200,000+ annotated radiographs (CheXpert, MIMIC-CXR)
Detections Fractures, pneumonia, pleural effusion, cardiomegaly, nodules
Output Heatmap overlay, bounding boxes, confidence scores
Capabilities Tissue segmentation, lesion detection, volumetric measurements Output Segmentation masks, annotated slices, measurement report

ECG Analysis

Model 1D CNN-LSTM hybrid classifier
Input Standard 12-lead ECG (10-second strips)
Classifications AFib, VTach, bradycardia, ST elevation, PVCs, normal sinus
Output Beat-by-beat annotation, rhythm classification, critical flags

4. API Reference

The Z-Ray backend exposes RESTful endpoints for programmatic access.

Authentication

POST /api/v1/auth/login
Content-Type: application/json

{
  "username": "admin",
  "password": "admin"
}

→ 200 { "token": "eyJhbG...", "expires_in": 3600 }

Submit Analysis

POST /api/v1/analysis
Authorization: Bearer <token>
Content-Type: multipart/form-data

Form fields:
  file      — medical image/data file
  type      — "xray" | "ecg"
  patient_id — optional patient identifier

→ 202 { "job_id": "a1b2c3", "status": "processing" }

Get Results

GET /api/v1/analysis/{job_id}
Authorization: Bearer <token>

→ 200 {
  "job_id": "a1b2c3",
  "status": "completed",
  "diagnosis": "Right distal radius fracture",
  "confidence": 0.963,
  "annotations": [ ... ],
  "created_at": "2026-03-16T14:23:00Z"
}

List History

GET /api/v1/analysis?limit=50&offset=0
Authorization: Bearer <token>

→ 200 { "total": 142, "results": [ ... ] }

7. Troubleshooting

Issue Solution
Login fails with correct credentials Clear browser cache and cookies, then retry
Upload timeout on large files Ensure file is within size limits; check network connection
"Model not loaded" error Run python scripts/download_models.py to re-download weights
Low confidence scores Ensure uploaded images meet quality standards (resolution, contrast)
Dashboard shows stale data Hard-refresh the browser (Ctrl+Shift+R)
GPU not detected Verify CUDA 11.8+ is installed: nvidia-smi

For additional support, contact support@zray-medical.com or open an issue on the project repository.
